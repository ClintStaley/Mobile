Kotlin Functional Programming

Write an Index.kt file that accepts as a commandline argument a text file and produces as output an index listing all words (sequences of alphanumeric, non-punctuation/whitespace, characters) that appear in the file, in sorted order, each followed by the line numbers on which they appear.

Example Input/Output

Here's an example input file, and corresponding output file. Note in particular that line numbers are 1-based, listed in order, and that and no line appears twice for a word, even if the word appears twice on the line.

Input:
100 bottles of beer on the wall
100 bottles of   beer!
if one of those bottles should happen to fall

99 bottles of beer on the wall

99 bottles of beer on the wall, 99 bottles of beer...

Output:
100: 1, 2
99: 5, 7
beer: 1, 2, 5, 7
bottles: 1, 2, 3, 5, 7
fall: 3
happen: 3
if: 3
of: 1, 2, 3, 5, 7
on: 1, 5, 7
one: 3
should: 3
the: 1, 5, 7
those: 3
to: 3
wall: 1, 5, 7

Assuming the input resides in file data.txt, and that Index.kt is compiled into Index.jar, this command should produce the example output:

java -jar Index.jar data.txt

Functional Coding Only

And here are the important restrictions, all intended to encourage functional thinking:

  • The entire program is in main, which has just one rather long, functional programming statement. You may have only this one statement, though it will span up to 5 lines. The model implementation, as a hint, uses these types and functions, among others: File, groupBy, forEach, flatMapIndexed, split, Regex

  • You may not declare any local variables in main. Parameters in lambdas passed to functions will be needed here and there, but prefer using "it" when possible. The model implementation has only two explicit uses of "->".

  • There is no need for type qualifiers (no ": String", etc) except for the main's parameter. All other variable types can be inferred, and you should let Kotlin do this wherever possible.
