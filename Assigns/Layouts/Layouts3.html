Introductory Android

(Layouts and Activities)

Overview

This and later exercises will draw on the introductory courses supplied at

https://developer.android.com/courses

and for the first homeworks, this course in particular:

https://developer.android.com/courses/android-basics-kotlin/course

Each of these courses is somewhat "code by the numbers" overview of Android dev topics. To use them effectively, note:

  • The videos are mostly just cute rah-rah, so you can skip those.

  • The codelab walkthroughs are where the serious content is. For these, actually write the code, don't just copy-paste it. And:

    • First, read through a portion of the presentation, up to an entire path depending on how good your memory is.

    • Then, *set aside the example code and presentation*, and write the code yourself, from memory as much as possible, checking the presentation only as absolutely necessary.

    • As you write each line, be sure you understand what it does and why.

  • A canned example like these codelabs is just a starting point.

    • Do deeper exploration of the classes used, via the API reference and other sources. Find out how they're organized, what their inheritance relationships are, etc. Bookmark useful points of the API reference.

    • Based on what you learned, explore "around" the code, trying different alternatives to see how they work.

    • I'll help with this by assigning required homework explorations surrounding the code in the walkthroughs, but learn also to do this for yourself.

Unit 3: Layouts: Display a Scrollable List

Much of this path centers on the sophisticated OO design surrounding RecyclerViews

  1. Review the developer.android.com | Guides | App Resources | Introduction and Localization.

  • Name at least 10 distinct properties of R, e.g. R.id, R.color, etc. How are such properties consistently referred to within layout and other resource files?

  • What tool/program generates the R class? What file does it produce? What kind of information does it add to that file?

  • What's an APK? Are resources like layouts, colors, etc. held in readable form in an APK?

  1. Adjust names and some functionality as you write the code, so as not to just cut/paste what's there. In particular

  • Choose less trite quotes. I used Bible and S&H. And change the name to Quotes not Affirmations. Drop the generic and unclear word "Item" in favor of "Quote" in class and property names.

  • Build the layout as described, including cards, etc. Snapshot this as part of your submission for this question. Then, adjust the layout of each image and its quote according to the attached snapshot

    • Place each quote inside the image, offset from the left edge and bottom

    • Put a background and border in gold and blue around each quote. Do this by creating a Shape Drawable describing the background and border and using it as the background for the TextView showing the quote.

    • Arrange the QuoteAdapter so that the font of the text alternates between normal and bold.

  1. Experiment by reversing the two args to Quote when supplying an image and a string. The IDE should note the error even though both args are strings. How does it do this?

  2. Use logging to instrument onCreateViewHolder and onBindViewHolder, so you can see how often they are called. Is there a difference in how many times each is called? Try doubling the number of quotes and images, by simply duplicating the ones you already have to make the data array twice as long. What happens to the number of calls of onCreateViewHolder and onBindViewHolder then? Why is this?
